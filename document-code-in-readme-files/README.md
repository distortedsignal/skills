# Document Code in README Files Skill

A skill for comprehensively documenting all code in README.md files

## Description

This skill performs thorough codebase analysis and generates comprehensive documentation for all README.md files in a project. It examines code structure, identifies design patterns, documents critical data structures, and creates detailed technical documentation.

## What it does

1. Finds all README.md files in the project
2. Analyzes the codebase in each directory
3. Updates each README with comprehensive documentation including:
   - Intent and purpose
   - Architecture diagrams
   - Directory structure
   - Critical data structures
   - Design patterns
   - Technology stack
   - Setup and development instructions
   - API documentation
   - Troubleshooting guides

## Usage

Simply invoke the skill when you need to document or update README files:

```
/document-readme-files
```

The skill will automatically:
- Discover all README.md files
- Analyze the codebase comprehensively
- Update each README with detailed documentation
- Track progress with todo lists

## When to use

- Starting work on a new project with poor documentation
- After major architectural changes
- Preparing for team handoffs
- During code reviews
- Creating open-source project documentation
- Onboarding new developers

## Output

Updates all README.md files in place with comprehensive documentation following a consistent structure including architecture diagrams, data structures, design patterns, and detailed setup instructions.
